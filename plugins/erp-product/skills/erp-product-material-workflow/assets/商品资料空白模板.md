# 商品资料

> 本文档用于维护 ERP 商品创建资料。请只填写真实、可确认的商品信息；不确定的字段保持空白。文件类字段填写相对路径，不填写 OSS URL。

## 0. 填写规则

1. 不需要手动查询系统 ID。分类、单位、供应商、区域、客户类型、配置项、技术参数、可选配置、配件单位等只填写名称，后续由系统工具解析真实 ID 并在提交草稿内部回填。模板中的 ID 列如存在，仅供系统回填，普通用户保持空白。
2. 文件类字段填写相对路径，路径相对于本 Markdown 文件所在目录，例如 `./图片/主图.jpg`、`./附件/证书.pdf`。
3. 不要填写 OSS URL、网络图片地址、文件 base64 或本机绝对路径。
4. 多个文件一行一个，或在表格中一行一个；不要把多个文件路径写在同一个单元格里。
5. 金额、重量、尺寸、数量、天数只写数字，不带单位。
6. 是否类字段只填写 `是`、`否` 或留空。
7. 日期统一填写 `YYYY-MM-DD`。
8. 状态字段支持 `上架`、`下架`、`作废`。不确定时留空；只有明确要求作废时才填写 `作废`。
9. 普通用户不要填写系统 ID；如果维护工具从后端查询后回填大整数 ID，应按文本保存，不使用科学计数法。
10. 创建真实商品前，需要由维护人确认资料真实有效，并确认允许上传资料包文件。

> 条件性必填说明：
> - 产品类型为 **整机** 时，必须填写产品等级、参考成本价 人民币、利润率 %，并在“8. 基础配置”填写至少一行整机配置；已填写的基础配置/技术参数必须补全配置值/参数值。
> - 产品类型为 **整机或配件** 时，必须填写包装长/宽/高 mm、包装费、包装重量 kg、净重 kg。
> - 适用范围为 **指定区域** 时，必须在“3. 适用区域”填写至少一行区域明细。

> 产品型号（可选字段，填写时硬格式校验）：`产品型号` 可留空；一旦填写，仅允许英文大小写字母、数字与空格，不能含首尾空格、中文或其他符号。

> 以下字段不作为硬阻塞必填项，按业务实际情况填写，留空不会阻塞创建：商品英文名称（可选）、Banner 图（可选）、包装方数（可选）。
